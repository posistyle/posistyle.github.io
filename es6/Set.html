# Set

# --> 类似数组的数据结构（集合），但是成员的值是唯一的，没有重复值

let s = new Set() //Set本身是一个构造函数

# 属性和方法

  ** 属性:

  Set.prototype.constructor: //构造函数，默认就是Set函数
  Set.prototype.size: //返回Set实例的成员总数

** 方法:
*1.操作方法
  Set.prototype.add(value): //添加某个值，返回是Set结构本身
  Set.prototype.delete(value): //删除某个值，返回一个布尔值
  Set.prototype.has(value): //表示是否包含某一个值，返回一个布尔值
  Set.prototype.clear(value):  //清除所有成员， 没有返回值

*2.遍历方法 （Set的遍历顺序就是插入顺序）
  Set.prototype.keys(): //返回键名的遍历器
  Set.prototype.values(): //返回键值的遍利器
  Set.prototype.entries(): //返回键值对的遍历器
  Set.prototype.forEach(): //使用回调函数遍历成员

  可以使用 for...of循环遍历

# 用法

1.去除数组,字符串重复成员的方法 [...new Set([array])] （array代表具有 iterable 接口的其他数据结构）

let set = new Set([1, 2, 3, 6, 3])
console.log([...set], set.size) //[ 1, 2, 3, 6 ] 4
  • 去重原理

内部判断俩个值是否相等，使用的算法类似与（===）,例外是向Set加入NaN时，认为NaN等于自身

  let setNaN = new Set()
  setNaN.add(NaN)
  setNaN.add(NaN)
  console.log(setNaN, setNaN.size) //Set { NaN } 1

2.Set的增删

let s1 = new Set()
s1.add({x:1}).add({x:1}).add(1).add([1]).add([1])
var obj = {x:1}
var arr = [1]
console.log(s1.has(obj), s1.has(1), s1.has(arr)) //false true false
console.log(s1.delete(arr), 'delete', s1.delete(1), s1.size) //false 'delete' true 4
console.log(Array.from(s1)) //将Set数据结构转化为数组 [ { x: 1 }, { x: 1 }, [ 1 ], [ 1 ] ]

3.遍历的应用

1.扩展运算符和Set结构结合实现去除数组的重复成员[...new Set(arr)]

2.数组的map和filter方法也可以间接用于 Set

let s2 = new Set([1,2,3])
let s3 = new Set([...s2].map( x => x * 2)) //Set { 2, 4, 6 }

3.实现并集，交集，差集

let a = new Set([1,2,3])
let b = new Set([4,2,5,3])
//并集
let union = new Set([...a, ...b])//Set { 1, 2, 3, 4, 5 }
//交集
let intersect = new Set([...a].filter(x => b.has(x))) //Set { 2, 3 }
//差集
let difference = new Set([...a].filter(x=> !b.has(x))) //Set { 1 }

4.遍历时同步改变原来Set结构

let s4 = new Set([1,2,3])
s3 = new Set([...s3].map(x=> x*2)) //Set { 4, 8, 12 }

# WeakSet

# --> 类似Set的数据结构，但是weakSet的成员只能是对象且WeakSet中的对象都是弱引用

const ws = new WeakSet() //作为构造函数，WeakSet 可以接受一个数组或类似数组的对象作为参数

# 方法

WeakSet.prototype.delete(value)：//清除 WeakSet 实例的指定成员。
WeakSet.prototype.has(value)：//返回一个布尔值，表示某个值是否在
